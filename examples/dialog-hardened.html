Implementation Details

Keyboard Navigation

  • Tab - Move focus to next focusable element (trapped within dialog)
  • Shift+Tab - Move focus to previous focusable element (trapped within dialog)
  • Escape - Close dialog and return focus to trigger button
  • Click Overlay - Close dialog

ARIA Attributes

  • role="dialog" - Identifies element as a dialog
  • aria-modal="true" - Indicates dialog is modal
  • aria-labelledby="..." - Links to dialog title
  • aria-describedby="..." - Links to optional description
  • aria-hidden="true/false" - Hidden from screen readers when closed

Accessibility Features

  • Focus trap: Tab/Shift+Tab cycle only within dialog
  • Focus restoration: Focus returns to trigger button when dialog closes
  • Initial focus: First focusable element receives focus when dialog opens
  • Body scroll lock: Background cannot be scrolled while dialog is open
  • Screen reader support: Dialog title and description announced
  • Escape key support: Press Escape to close dialog

CSP Compliance

  • No inline event handlers (onclick, onkeydown, etc.)
  • No inline script blocks
  • All behavior in external ES module
  • CSS-only styling (no style attribute)
  • Compatible with strict CSP: script-src 'self'
